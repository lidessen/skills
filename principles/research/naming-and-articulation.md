# Research — Naming and Articulation

**Disposition:** no-proposal
**Scope:** Determine whether recurrent project naming, definition, explanation,
and documentation placement require a new Sequence entry or a downstream
method; this may change how shared concepts acquire durable names and where
their explanations live.
**Source limitations:** The Dao De Jing sources establish a philosophical
constraint on names, not a complete technical-documentation procedure. Project
evidence is local to this collection.

> This record is cited, revisable research. It owns no P-ID, cannot redefine an
> interpretation, and does not propose a Sequence change by itself.

## Question

When a project needs to name itself or a concept, define the term, explain it,
and choose where that explanation belongs, what method makes the name useful
without mistaking a label for the object or multiplying unnecessary taxonomy?

## Distinctions

- A **name** is a compact handle; an **operative definition** states the
  decision boundary the handle preserves; an **explanation** reconnects that
  boundary to concrete evidence and a reader's next action. They are not
  interchangeable prose lengths.
- Naming a shared, decision-changing relation differs from branding, stylistic
  copyediting, a one-off variable name, or an exhaustive glossary.
- This inquiry does not treat a Dao De Jing passage as a hidden new Sequence
  rule. It tests whether current P-IDs already express its decision consequences.

## Evidence

The opening of the [*Dao De Jing*, chapter 1](https://ctext.org/text.pl?if=gb&node=11592&show=parallel)
places naming between two facts relevant here: names make distinctions among
things possible, yet a nameable label is not the final or exhaustive reality.
For a project method, that supports treating a name as a deliberate operating
handle rather than a claim to capture an object's whole essence.

Chapter 25 says that where the name is unknown, one may give a provisional
designation — “强为之名” — in the [primary text](https://ctext.org/dao-de-jing/zh).
That supports a visible provisional-name state when action cannot wait for final
understanding; it does not license arbitrary branding.

Chapter 32's [“始制有名……知止”](https://ctext.org/text.pl?if=gb&node=11623&show=parallel)
connects naming with a stopping boundary. In this project, that means a new
term or glossary entry must change a future distinction, recovery path, or
action; otherwise the correct action is to stop naming.

The collection already needed exact status vocabulary: the verified organization
probe distinguishes [decision 008's `adopted and verified` status](../../design/decisions/008-project-first-work-cell-interaction.md)
from decisions 009–012's `accepted for implementation` / `verified for human
approval` statuses. Collapsing those words would incorrectly promote claims
into shared fact, as the [organization evaluation](../../regeneration/evaluations/2026-07-10-bounded-adaptive-organization-probe.md)
records.

## Existing-sequence coverage

- **P06:** already prevents treating a visible label as the relation that
  governs the object; it supplies the core `name ≠ essence` gate.
- **P07:** already requires a term to arise from a concrete situation and return
  to an intelligible concrete whole, rather than merely relabelling a preference.
- **P16:** already asks whether the actual reader can act from the expression.
- **P15:** already supplies the stop rule: only retain distinctions that close a
  present contradiction while preserving hard constraints.
- **P12 / P13 / P14:** govern respectively what must survive for a later actor,
  when a definition becomes a verified shared claim, and why an index or
  generated glossary is not automatically authoritative.

## Possible decision delta

Existing principles justify a reusable **downstream naming-and-articulation
method**: it can decide whether a term is needed, test candidate names against a
concrete relation and contrast, write the smallest definition/explanation, and
place it beside its owning source. This changes repeated project actions without
changing the Sequence.

## Strongest no-proposal case

No new P-ID is warranted. The distinction between a name and the relation it
serves is P06; the movement from actual object to usable expression is P07/P16;
the stopping condition is P15. A candidate about “names” would restate that
combination while adding permanent semantic surface.

## Disposition and next evidence

**Disposition:** `no-proposal`

Create a skill only if a behavior probe shows that agents can use this combined
method to produce a decision-changing name/definition/explanation and decline
one-off or branding-only requests. A failure that cannot be reduced to these
P-IDs would reopen this inquiry.
