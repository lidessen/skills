# Development log

This directory contains curated checkpoint summaries: the target, implemented
transition, verification references, drift judgment, method feedback, and next
adjustment for a completed development movement.

The log is a navigational synthesis, not primary evidence and not a universal
task journal. Raw runtime records stay with their creating system, behavior
evaluations belong in [`regeneration/evaluations/`](../regeneration/evaluations/),
decision-changing observations and corrections belong in the
[`chronicle/`](../chronicle/), and accepted design belongs in
[`design/`](../design/). Add a checkpoint only when its method feedback or
handoff remains useful beyond the immediate task.
