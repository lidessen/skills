# 013 — Naming and Articulation Is a Bounded Method

**Status:** verified for human approval
**Date:** 2026-07-10

## Context

Project names, concept names, status words, definitions, explanations, and
documentation placement repeatedly determine what later people and agents can
perceive and do. They are currently distributed among design decisions, skills,
and prose conventions. The current collection has no owner for the preceding
judgment: whether a name is needed, what actual relation it must preserve, how
it differs from nearby terms, and where the smallest explanation belongs.

This is not generic writing. The archived technical-article method owns an
argumentative article, while [`skill-engineering`](../../skills/skill-engineering/SKILL.md)
owns the expression architecture of a skill. Neither owns shared project
vocabulary or the boundary between a local explanation and a durable term.

The source-bound [naming research](../../principles/research/naming-and-articulation.md)
finds that current P06/P07/P16/P15 already govern this decision; no Sequence
candidate is proposed.

## Decision

Add `skills/naming-and-articulation` as a low-frequency method for this one
judgment:

> Given a concrete object, audience, and blocked decision, what is the smallest
> name, operative definition, explanation, and source location that makes the
> relevant relation usable without treating the label as essence or creating a
> needless taxonomy?

Its form stack is deliberately narrow:

| Role | Owner |
|---|---|
| Naming and articulation method | `naming-and-articulation` Skill |
| Durable definition, if a shared boundary must survive | owning design/skill/source artifact |
| Public overview or generated concept index, if later useful | rebuildable projection linked to its source |
| Semantic root or P-ID meaning | Sequence and principle-cultivation only |
| Acceptance of a durable shared term | designated human/host authority |

The Skill does not create a universal glossary, brand strategy, terminology
committee, new semantic source, or documentation tree. A term remains local or
unwritten unless its absence would change a shared decision, recovery path,
authority boundary, or repeated action.

## Method

P06 leads: recover the governing relation before choosing a label. P07 moves
from concrete evidence through the necessary determinations and back to a term
usable in the actual situation. P16 tests whether the intended reader can make
the next distinction from the expression. P15 stops adding names, layers, and
documentation when they no longer change a decision.

The result has separate layers only when each is needed:

1. **Name** — a compact, revisable handle; it does not claim exhaustive essence.
2. **Operative definition** — one decision boundary: what relation it names,
   what it excludes, and the contrast that would reveal misuse.
3. **Explanation** — evidence and context showing why the boundary matters to a
   specific reader; examples, metaphor, and translations remain subordinate.
4. **Placement** — the definition lives beside the source whose future decision
   it changes: Sequence interpretations for a P-ID, design for architecture,
   a skill's direct references for agent method, or local code/doc context for
   an implementation detail. A central index is only a projection.

## Acceptance

- The Skill has a distinct repeated naming/definition/explanation trigger and
  routes article writing, one-off prose, semantic changes, and skill-expression
  engineering to their actual owners.
- It names P06 as Primary with P07/P16/P15 as distinct supports, carries the
  matching portable Sequence snapshot, and has no independent doctrine.
- An action probe turns the current status/organization ambiguity into a
  source-bound naming record and correctly places its explanation.
- A boundary probe declines branding or a one-off sentence that creates no
  shared decision; an authority probe keeps a generated index subordinate.
- No global glossary or new Sequence line is created merely because a term is
  useful.

## Implementation evidence

The [naming-and-articulation action and boundary probe](../../regeneration/evaluations/2026-07-10-naming-and-articulation-probe.md)
retains a constrained successful run: it chose and placed an operative
definition for the organization term while declining a durable artifact for
one-off prose. The result is behavior evidence, not proof of general naming
quality or human adoption.
