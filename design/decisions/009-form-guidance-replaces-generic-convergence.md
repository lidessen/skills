# 009 — Form Guidance Is a Specialized Form-Selection Method

**Status:** accepted for implementation
**Date:** 2026-07-10

## Context

The externally installed `task-convergence` starts substantial work with a
generic design–implement–verify–feedback loop. The project’s
[`disciplined-development`](../../skills/disciplined-development/SKILL.md)
already supplies the closer default layer for ordinary work: evidence, concrete
contradiction, minimum transition, and falsifiable verification. Neither should
be conflated with the rarer judgment of deciding whether a capability should
exist as a skill, durable decision record, runtime capability, projection,
bounded campaign, or no new form.

Recent Work Cell work demonstrated that missing judgment. The runtime's exact
JSON contract was correct, yet daily project use required a thin project
projection rather than a new scheduler or a rewritten core; see the [accepted
project probe evaluation](../../regeneration/evaluations/2026-07-10-work-cell-project-probe.md).
The distinction was recovered only by manually combining artifact role,
authority, recurrence, and action-form reasoning.

## Decision

Add `skills/form-guidance` as this collection's independently installable,
low-frequency form-selection method. It owns one decision: select the smallest
truthful form stack that enables a recurring actor's next judgment while
retaining source, authority, and inheritance boundaries.

It does not replace implementation, verification, or feedback inside the form
that it selects. Those loops remain with their actual owners: for example,
[`skill-engineering`](../../skills/skill-engineering/SKILL.md) for agent-facing
methods, [`artifact-organization`](../../skills/artifact-organization/SKILL.md)
for authority-and-lifetime layout, and [Work Cell](../../packages/work-cell/README.md)
for bounded execution evidence.

It is not a default task entrypoint and does not replace `task-convergence` or
`disciplined-development`. It activates only when a concrete form contradiction
is evidenced. It selects a carrier and names the owner of the selected form's
domain method, then stops; it cannot define that method's strategy, doctrine,
or roadmap. A separate replacement for generic convergence must be designed
from the disciplined-development practice layer, then compared with the global
installed entry before any installation decision.

## Sequence expression

- **Primary P16:** form is valid only when the actual actor can take the next
  action from it.
- **P04:** select the form mismatch whose repair changes downstream work.
- **P14:** generated summaries, dashboards, and plans remain projections unless
  they have a named source and independently governed promotion.
- **P15:** prefer the least new machinery that closes the form contradiction.

## Acceptance

- The Skill states a distinct trigger, action gap, form comparison, authority
  boundary, and routing boundary.
- It carries a generated portable Sequence snapshot for P16/P04/P14/P15.
- Action, boundary, authority, and context probes show that it selects forms
  only when a concrete form contradiction is present.
- The selected form's domain owner is explicit, and a handoff probe shows that
  form-guidance does not write the selected domain's doctrine or workflow.

## Implementation evidence

The [practice-cycle and form-guidance probes](../../regeneration/evaluations/2026-07-10-practice-cycle-and-form-guidance-probes.md)
retain a live handoff result and a one-off/no-form boundary result. They do not
establish universal improvement or replacement attribution.
