## Action gap

- **Target action gap:** an agent facing a substantial project repeatedly has
  to reconstruct how the project works, how to divide that cognition into
  coherent bounded investigations, how to preserve the useful result for later
  agents, and how to refresh only the relations affected by a change.
- **Actor and conditions:** an ordinary coding agent operating in a target
  repository, optionally using Work Cell or another sub-agent carrier, without
  access to the conversations that produced this method.
- **Concrete evidence:** the [Shilu bootstrap and refresh probe](../../../regeneration/evaluations/2026-07-16-project-cognition-bootstrap-and-refresh.md)
  showed a material decision gain over one whole-project model, required local
  repartition after one packet failed, and used a different changed-relation
  partition for selective refresh.
- **Improvement observation:** from the installed Skill and a named source
  revision, a fresh agent chooses a project-specific decomposition, produces a
  source-linked and reconstructible carrier, preserves uncertainty and authority
  boundaries, and later refreshes only source-supported affected facets. It
  declines persistence when the task is one-off.

## Selected expression

**Primary P05 — concrete analysis.** The Skill's stable gate is not “create
these standard facets”; it is selecting a cognition and partition shape from
the actual project, question, history, constraints, actors, and change field.

**Supporting P12 — external memory and inheritance.** Persistence is justified
only when a later actor can recover a decision-relevant working model without
replaying the original session. It determines what must survive and what may be
regenerated or forgotten.

**Supporting P13 — claims require traceable verification and submission.** A
Cell output, synthesis, or plausible relation remains a claim until source
anchors and a designated verifier support its admission.

**Supporting P14 — reconstructible state is a projection.** The cognition
index and facets improve access but cannot become project fact authority merely
because later agents read them first.

**Current task lead:** P16 from `skill-engineering` leads the present expression
work; it is not part of the target Skill's portable lineage.

## Existing owners and boundary

- `code-review` owns an evidence-linked working model and semantic partition
  for one proposed change. It does not own a reusable cross-task cognition
  projection or its lifecycle.
- `context-engineering` owns when and how selected source information reaches an
  agent. It does not author project cognition or choose semantic partitions.
- Work Cell owns bounded execution and evidence; orchestration owns release and
  settlement. Neither owns domain decomposition, synthesis, or acceptance.
- `artifact-organization` remains the owner when a target project has a
  contested source, authority, or durable path rather than a cognition-method
  problem.

An independently installable Skill is the smallest truthful form because the
gap is a repeatable activation-time judgment. A local instruction is not
portable; a reference alone does not route the action; a runtime is unnecessary
for semantic selection; and a projection without a method only preserves one
instance.

## Non-scope

The Skill must not define a universal ontology, fixed facet taxonomy, Work Cell
schema, concurrency policy, database, retrieval service, source authority,
project acceptance rule, code-review verdict, or mandatory durable memory. It
may use those existing capabilities when the target project provides them, but
it cannot make them prerequisites.
