**Status:** pre-selection. This map is the analytical middle term between the
concrete inventories and a target architecture. It records candidate functions,
not future skill names.

## Next selection test

For each row, the target architecture must answer all four questions:

1. Does this function own a decision that no retained function already owns?
2. What concrete evidence proves that a user reaches for it repeatedly across
   contexts?
3. Can it be expressed with one Primary P-ID and no more than three supporting
   P-IDs without inventing doctrine?
4. Is a distinct skill the minimum valid form, or would a command, reference,
   or project-local adapter express it more truthfully?
