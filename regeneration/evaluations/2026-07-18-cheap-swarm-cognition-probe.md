## Fixed question and withheld ledger

The probe asked what a later agent must recover to continue the staged Atthis
workbench and general cognition experiment safely. Before model execution, the
caller fixed nine load-bearing relations and five required gaps against the
named source tree. The execution Cells did not receive that ledger.

The fixed field covered:

- Atthis identity versus repository, alias, and path;
- the workbench resolver's mechanical and authority boundary;
- Sequence, Skill, Work Cell, Cognition, projection, and human authority;
- implemented cognition formation, maintenance, and integrity;
- unimplemented filtering, coalescing, background, retention, authentication,
  and behavioral-use proof;
- the Cognition Cell adapter and Swarm responsibility split;
- public documentation and CI as bounded projections and evidence; and
- empty workbench ownership boundaries that must not be populated by inference.

The compared forms were one whole-field Cell and four semantically partitioned
Cells: identity/entry, cognition/authority, execution boundary, and
delivery/verification. All were read-only, command-free, and required the same
`submit_cognition` terminal contract. Raw evidence remains under the ignored
`.work-cell/cognition-swarm-probe/` directory; the run IDs and observations
below are the durable projection.

### Matched Flash rerun

The repository index had advanced after the first comparison, so the rerun did
not read the current checkout. It reconstructed staged tree
`25c339ebc42f6dd9440b4368b3ed62cbb1d5c647` from the Git object database and
executed that tree's Work Cell source against the unchanged four-Cell manifest,
worker-visible packet, read paths, terminal contract, concurrency, step limit,
and duration boundary. The only deliberate profile change from the original
four-Cell Kimi run was the explicit DeepSeek provider/model route. This supports
a whole-execution-profile comparison, not a bare-model claim.

Both serial repetitions settled all four Cells. Together they produced eight of
eight passed records in 43.4–47.1 seconds per Swarm, compared with one passed and
three right-censored records at 300 seconds for the matched Kimi run. The Flash
runs recorded approximate costs of `$0.02301` and `$0.02550`.

Settlement latency was repeatable, but token use was not: totals ranged from
313,484 to 425,517, and the delivery Cell alone ranged from 125,004 to 216,953.
The unchanged 150,000-token estimate understated the two actual totals by 109%
and 184%. This reinforces that `estimatedTokens` is an audit estimate rather
than an execution limit and that one fast run is not a stable token forecast.

Source checking against the withheld ledger found full recovery of eight of nine
load-bearing relations and all five required gaps in both repetitions. L3 was
only partial: the packets observed that Cognition and Work Cell cannot acquire
Sequence authority, but never reconnected the complete relation that Sequence is
the sole semantic root and Skills are its downstream expressions.

Protocol success also concealed concrete semantic defects:

- repetition 1's delivery Cell reported five cognition tests while its own
  evidence listed the six tests present in the fixture;
- repetition 2's identity conclusion assigned stable project-identity ownership
  to the resolver, while decision 038 assigns it to `config/projects.json` and
  limits `scripts/atthis.py` to mechanical actions; and
- repetition 2's delivery conclusion said CI lacked a workbench-resolution test,
  while the frozen workflow explicitly ran `python3 scripts/test-atthis.py`; a
  later claim in the same submission contradicted that conclusion.

The source check was evaluator-visible rather than blind, so it supports defect
identification, not a comparative semantic-quality winner. The bounded result is
that this Flash execution profile is a strong default candidate for read-only,
source-partitioned cognition Swarms: it settled much more reliably and quickly
than the matched Kimi profile. Its records still require cross-packet
reconciliation and independent source verification before cognition admission.

The reconciliation Cell did not recover complete L3, did not expose the packet's
stable-identity ownership correction or delivery/CI contradiction, and claimed
that no decision-changing packet conflict existed. It also added a false
limitation that the preceding work had not satisfied its terminal-tool contract,
although the retained Swarm index records all four Cells as passed. Its
`usable-proposal` verdict was therefore rejected.

The initial verifier was not an independent semantic gate in practice. It
recommended `admit`, credited source facts absent from the candidate as candidate
coverage, labelled partial findings as covered, and reported correct claims as
defects. A matched prompt treatment separated candidate coverage from source
truth and changed the recommendation to `revise`; it found L5, G4, and several
omissions, but still credited absent L3, G1, and G3 content and incorrectly
treated system gaps as packet conflicts. This is development-case evidence for
the prompt treatment, not held-out verifier capability.

The next treatment changed execution form rather than adding more instructions
to one attention surface. Three Cells independently owned candidate-only ledger
coverage, source truth, and packet-lineage/internal consistency. The coverage
Cell correctly rejected admission: it found L3 and L7 partial, L5 and G1-G4
missing, and did not use repository truth to fill the candidate. The source-truth
Cell nevertheless missed the false terminal-settlement limitation, while the
lineage Cell missed unacknowledged packet corrections and attributed content to
candidate sections where it did not exist. Their mechanical `pass` records do
not repair those semantic failures.

A fail-closed host can use any negative verifier result to block admission, so
the orthogonal form prevented the unsafe `admit` outcome. It is safer, not yet a
validated verifier. Source-truth and lineage roles need a retained case that
they can actually discriminate before their positive results have standing.
The runtime should therefore remain generic; verifier role separation and veto
policy belong to the cognition domain method or harness.

For the tested second repetition, the potentially operational path—four local
Cells, one reconciliation Cell, and the orthogonal verifier Swarm—used 933,848
tokens and about `$0.06467`. The verifier split itself used 40–47% fewer tokens
than either monolithic verifier attempt, but semantic closure still cost more
than local formation. Cheap model calls do not make admitted cognition cheap:
reconciliation and verification are first-class work, not negligible tail
steps.

No cognition proposal from this follow-up was admitted. The planned inheritance
comparison remains blocked until a source-truth and lineage gate catches the
retained counterexamples or a human/source-backed repair produces a candidate
that can be submitted deliberately.

## Source-checked quality comparison

The whole-field result covered all nine fixed relations and five required gaps.
It also found two real integration drifts outside the fixed ledger:

1. `packages/work-cell/src/index.ts` exported the cognition adapter through the
   main barrel while [the package's generic-core rule](../../packages/work-cell/README.md#generic-core-promotion-rule)
   described domain carriers as explicit adapter entry points; and
2. [the architecture overview](../../design/DESIGN.md#modules) had not yet
   incorporated accepted decisions 038 and 039 as Workbench and Cognition
   modules.

The final local packet set also covered the whole fixed ledger after
reconciliation. It improved precision in several places:

- the cognition adapter validates request and output shape, but its statements
  against semantic overreach are instructions and acceptance text rather than
  proof that a schema-valid proposal is truthful;
- 256-Cell evidence is deterministic scheduling evidence, not proof of 256 live
  provider calls or safe shared mutation;
- origin-less repositories are discoverable by the workbench but current
  registration still requires an origin; and
- Kimi `k3` and DeepSeek Flash showed materially different fitness for the same
  narrow multi-file tool task.

The packet set was not safe to concatenate or vote. The delivery packet claimed
that CI had no cognition formation/admission/catalog test, no cross-package
cognition adapter test, and that public surfaces were synchronized. Source
inspection rejected those claims: `.github/workflows/verify.yml` runs the
package test suites, `packages/cognition/test/cognition.test.ts` exercises the
formation and maintenance contracts, and
`packages/work-cell/test/cognition-adapter.test.ts` exercises the adapter. The
barrel/documentation mismatch also disproved the claim of complete public
synchronization.

This is the useful reaction: bounded Cells produced complementary local
precision and one Cell exposed a mistaken inference that another packet's
evidence could refute. It becomes cognition only after a separate actor
reconnects boundaries and checks source. Agreement, terminal success, and
structured output would not have corrected the error.

## Revised judgment

Cheap Swarm is supported as a conditional cognition formation carrier when:

- deterministic evidence first narrows the changed field;
- each Cell has one coherent question and few decision-bearing sources;
- differentiation changes evidence scope or method;
- failed regions can be narrowed without rerunning successful siblings;
- reconciliation receives every boundary obligation and can reopen source; and
- provider/model execution profiles are calibrated for that packet shape.

It is not supported as a default memory stage or as an economical replacement
for one coherent Cell. Current Kimi `k3` behavior suggests that file count,
semantic breadth, tool-loop steps, and tail latency are all part of a Cell's
stable envelope; a token estimate alone is insufficient. Current Work Cell
routing also cannot recover a Cell cheaply after a provider request consumes
the whole Cell timeout.

## What this did not prove

The probe compared execution forms over one cognition question. It did not
register a domain formation scheme, retain the reports as multi-stage cognition,
admit them, retrieve them in a later task, or show that inherited cognition
changes that later decision correctly. It therefore does not close decision
039's behavioral inheritance gate.

The next smallest practice is one narrower end-to-end use: define a project-
local formation scheme outside the kernel, retain the four source-checked local
models only as proposals, reconcile them into one proposed routing model, and
ask a later fresh Agent one withheld cross-boundary question with and without
that admitted inheritance. Measure decision delta, retrieval work, unsupported
inheritance, and refresh cost before adding any background capture or graph
projection.
