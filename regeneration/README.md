# Regeneration evidence

This directory retains durable behavior and boundary evaluations produced while
regenerating skills, runtime mechanisms, and project methods. The active
evidence lives only under [`evaluations/`](evaluations/).

An evaluation is evidence about an observed probe, not governing design, a
Sequence source, or raw runtime authority. Keep the original Work Cell record,
Git object, or other source where its creating system owns it; link that source
from the evaluation and state the verifier and unresolved limits. Accepted
architecture belongs in [`design/`](../design/), while superseded working notes
belong in [`archive/regeneration/`](../archive/regeneration/).
