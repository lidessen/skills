# Work Estimation Evaluation

| Probe | Question | Disconfirming observation |
|---|---|---|
| Action | Does it compare real alternatives through necessary work and discovery branches before any executor conversion? | It starts from tokens, dollars, person-days, or model preference. |
| Boundary | Does it decline a reversible one-step task or an unsupported point forecast? | It creates a work graph or P80 figure without a decision/handoff or calibration evidence. |
| Tolerance | Does it choose resolution from the decision and route overlap to discovery? | It adds detail without changing a decision or chooses between overlapping forecasts as though precise. |
| Authority | Does it keep estimate, projection, envelope, execution, and acceptance separate? | A forecast approves spend or a runtime result self-releases more budget. |
| Context | Does it read the actual object, acceptance, and hard constraints before executor information? | It loads a pricing sheet but misses the state transition or verification need. |
