# Capability Case

Use this artifact to turn retained evaluation evidence into one reviewable
allocation claim. Keep it close to the owning project's evaluation records. It
is candidate evidence until the named authority accepts it.

```text
Capability Case ID and revision:
Status: probe | candidate | accepted | superseded
Allocation decision:
Bounded capability claim:

Execution profile identity:
  model and actual provider or plan:
  harness and version:
  prompt, skill, and context policy revision:
  tools and permissions:
  execution and fallback policy:

Task population and excluded work:
Case selection and acceptance provenance:
Fixture and source revisions:
Capability dimensions exercised:
Named material failure classes:

Repeated run evidence:
  exact record links:
  pass, unsettled, retry, and intervention observations:
  within-profile variation:
  between-profile difference, when compared:
  actual served identities:
  latency and usage:
  observed money or plan consumption, when available:

Blind or independent judgment evidence:
Contrary evidence and retained failures:
Strongest alternative explanation:
Prompting observations and treatment hypotheses:
  discovery cases that exposed the hypothesis:
  one changed profile member and fixed comparison members:
  separate confirmation cases and result:
Confidence basis, without a scalar intelligence score:

Claim scope and prohibited extrapolations:
Observation that would reopen or reject the claim:
Expiry or review trigger:
Named admission authority and decision:
```

## Admission questions

- Do the cases represent the work this claim may govern, or only a convenient
  proxy?
- Can every claimed condition be traced to an exact run and source revision?
- Is within-profile variation smaller and less material than the claimed
  between-profile difference?
- Were failures and interventions retained with the same weight as successes?
- Could provider routing, harness behavior, or context delivery explain the
  result better than model capability?
- Did any evaluation case teach a prompt change? If so, was it removed from the
  held-out confirmation claim?
- Would accepting the claim change a real allocation decision?

If the final answer to any evidence question is unknown, keep the status
`probe` or `candidate`. Human acceptance records the bounded claim; it does not
make excluded work or future model revisions equivalent.
