# show — Display + interpret (read-only)

`/writing-profile show` 是测试完之后**看结果**的地方。

**两件事并列**：展示（给出坐标和标签）+ 解读（说清楚这个画像在写作里意味着什么、和哪些作家近或远、容易滑向什么陷阱）。不只 dump 坐标——用户敲 `show` 想看的是画像讲了一个什么"关于自己的故事"。

不触发副作用（不启动测试、不改 state、不 finalize）。

## 决策

读 `~/.claude/writing-profile/` 下的状态：

### A. 什么都没有

`state.md` 不存在 → 告诉用户画像尚未建立，提示：
> 尚未开始。用 `/writing-profile` 开始建画像（约 30 分钟，可随时中断）。

### B. 画像已完成（profile.md 存在且未 stale）

三段结构：**一行类型** → **坐标** → **解读**。布局如下：

```
# 你是：ENTP-R · 杂文家
作家谱系：王小波 × 鲁迅杂文

## 坐标
L1 认知启动：
  - S/N: +7 (直觉强)     W1 
  - T/F: +4 (理性)       W2
L2 文本姿态：
  - E/I: +5 (外倾)       W3
  - J/P: -3 (发散)       W4
L3 审美偏好：
  - D/L: +4 (致密)       W5
  - G/H: -8 (强反讽)     W6
  - C_V: -3 (白话)       W7   [置信度 high]

## 解读
写 2-4 段话说清楚这几件事：

1. **这个画像是谁**——一句话把 type 翻译成写作里的人，不是 MBTI 翻译本。如："你偏概念、偏逻辑、对读者说话、讨厌稀释、靠反讽做事、用白话——这是一个擅长在白话里藏机锋的杂文人。"
2. **最强和最弱的轴**——挑 |score| 最大的 2-3 个维度，说这些强信号在稿子里会怎么显（强 G_H 反讽 → 容易写出带毒的比喻；强 D_L 致密 → 段落容易超长；弱 F → 情绪表达常被推理顶掉）。
3. **容易滑向的陷阱**——画像的阴面。强反讽的人容易"抖机灵过头"，强致密的人容易"读者累"，强 N 的人容易"悬空"。给 1-2 个警戒信号。
4. **和作家锚点的关系**——profile.md 里已经有最近的作家名字；这里用一句话点出"像 A 的地方"和"不像 A 的地方"，避免用户把自己等同于某个作家。

不要把画像吹成身份标签。这是当前偏好，不是终点。

## 可复制分享（type 码）
ENTP-R · 杂文家 (王小波 × 鲁迅杂文)

## 下一步
- `/writing-profile W<N>` — 重测单维度（觉得某轴不准）
- `/writing-profile retest` — 全部重测
- `/writing-profile review <path>` — 用画像审一篇文档
```

`## 解读` 的措辞要像对着人讲，不是填表。confidence 低的维度在解读里显式标出（"这条置信度 medium，下次可以 retest W<N> 复核"），别当铁板一块讲。

完整画像（机械反 AI 清单、作家锚点图、维度详解）在 `~/.claude/writing-profile/profile.md`，提一句指向即可，不要把文件整块 dump 进对话。

### C. profile.md 存在但 stale（某维度已 retest）

告知："画像已标记为 stale（W3 重测后尚未 finalize）。执行 `/writing-profile` 自动重新合成。"
再简要展示旧 profile 的 type 标签做参考。

### D. 测试进行中

没有 profile.md，state.md 存在，有些维度完成有些未完：

```
# 写作画像 · 进度
创建：2026-04-18 14:30
最近更新：2026-04-18 15:45

## 维度进度
- [x] W1 感官-直觉 (S/N)    → N (+7, high)
- [x] W2 理性-感性 (T/F)    → T (+4, medium)
- [~] W3 外倾-内倾 (E/I)    [进行中, 5/12, 当前 I (-3)]
- [ ] W4 收束-发散 (J/P)    未开始
- [ ] W5 致密-疏朗 (D/L)    未开始
- [ ] W6 严肃-反讽 (G/H)    未开始
- [-] W7 文言-白话 (C/V)    已跳过 [classical_exposure: low]

## 已确认坐标（部分）
S_N: +7 (high)
T_F: +4 (medium)

## 下一步
`/writing-profile` 继续（会自动接 W3 或跑下一个未开始维度）
```

### E. 计划阶段刚完成但未跑任何维度

展示 plan 阶段的 meta-intake 结果和计划：

```
# 写作画像 · 已完成计划
计划生成：2026-04-18 14:30
Meta-intake: [5 题答案摘要]
Calibration: [列出标记的 flags]

## 下一步
`/writing-profile` 开始 W1
```

## 保持的性质

- **只读**：不启动任何测试、不改 state、不 finalize
- **不重复 dispatch 的工作**：如果用户想推进，让他们 `/writing-profile`
- **信息完整但不冗长**：完整细节可指向具体文件

## 问题处理

- 如果发现 state.md 和 units/ 状态不一致（某 unit 文件存在但 state 没标记，或反之），在输出里加一条 "⚠️ 数据不一致，建议 `/writing-profile retest` 后重来（或 `rm -rf ~/.claude/writing-profile/` 彻底清空）"
- 如果 profile.md 的时间早于最近的 unit 更新，标 stale 并提示
