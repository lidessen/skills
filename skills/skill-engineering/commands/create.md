# Create an Agent Skill

Create a skill from an observed recurring action gap, not from a name or a
template.

1. Read the target project's governing instructions and relevant source
   artifacts. Resolve the Sequence through the source resolver in SKILL.md;
   use the packaged snapshot when the host does not supply one. State the
   agent, its required action, the recurring failure or opportunity, and the
   evidence that makes this a real gap.
2. Read references/expression-team.md. Select the target skill's one Primary
   P-ID and up to three supports, then read only their interpretations. If no
   selection has a decision delta, do not create a skill.
3. Test whether a smaller form is sufficient: an existing skill, command,
   reference, project instruction, or tool adapter. A new skill needs all of:
   a repeatable trigger, a distinct judgment, and a durable artifact or gate.
4. Build the smallest target directory that expresses the decision. Keep
   trigger classification in frontmatter, the work loop in SKILL.md, and only
   genuinely conditional detail in direct supporting files. Do not put an
   explanatory second canon into the target skill. When the domain is abstract,
   add `references/concepts.md` for domain doctrine per
   references/expression-layers.md — not skill-authoring rules.
5. Read references/evaluation.md and define the action, boundary, and context
   probes before calling the new skill complete. Run safe probes now when the
   task permits; otherwise leave their exact commands and acceptance evidence
   as an explicit follow-up.

Deliver the target action, its expression-team selection, the chosen form and
rejected smaller alternatives, changed paths, and probe evidence or remaining
test gap.
