# Target Expression Team

Use this record before creating or materially rewriting a target skill. It is a
design selection for one expression, not a principle-review committee.

## Required record

```text
Target action gap:
Actor and real operating conditions:
Concrete source evidence:
Observation that would show improvement:

Primary P-ID:
  decision this P-ID owns here:
Supporting P-IDs (0–3):
  distinct decision contributed by each:
Current task lead P-ID:

Alternate candidate trial (optional, host-nominated only):
  candidate record:
  baseline choice without candidate:
  different choice proposed:
  disconfirming observation:

Existing owner checked:
Why a skill is smaller or more truthful than alternatives:
Non-scope and routing boundary:
```

## Selection rules

- Resolve the Sequence through the source resolver in SKILL.md before selecting
  P-IDs, then read only the selected interpretations. Record whether the host,
  packaged baseline, or temporary refresh supplied the lineage.
- Select one Primary: it owns the skill's stable decision gate, not merely its
  most attractive slogan.
- Add a support only when it contributes a distinct recurring decision. Remove
  a support that only restates the Primary or a local implementation choice.
- Let the current lead differ from the Primary only when the live task's
  principal contradiction requires it. Do not invent co-primary doctrine.
- If two target skills own the same action and evidence, merge them or keep the
  existing owner. A different name, directory, or model is not a new gate.
- If the evidence suggests a missing cross-context principle, stop selection
  and use principle-cultivation; do not hide it in a target skill.
- A host-nominated alternate is an optional trial outside the expression team.
  Use at most one, never list it as Primary or Supporting, and do not include it
  in a portable Sequence snapshot. Retain the observed outcome in its candidate
  record through principle cultivation.
