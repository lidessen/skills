# Verify Context Delivery

Verify the actual path and action, not configuration plausibility.

1. Read `references/delivery-capability-map.md` and
   `references/evaluation.md`. State the runtime, target action, source,
   declared delivery path, and disconfirming observation.
2. Run a real task through the agent's ordinary entry surface without pasting
   the expected conclusion. Observe whether the named source or projection
   reached the task at the required moment.
3. Check the action condition independently. A trace can prove loading without
   proving correct use; a correct answer can be compatible with the source
   without proving it was loaded.
4. Run a nearby boundary task when unnecessary loading or over-triggering is a
   material risk.
5. Use a controlled baseline only when the claim is comparative improvement.
   Without it, report delivery/use compatibility and leave attribution open.
6. Retain evidence only when a later actor must recover the result. Record
   unknown paths as unknown rather than creating a durable artifact by default.
