# Design Context Delivery

Design a delivery path only after the target action and source are known.

1. Read the project instructions, accepted design, and the named source. State
   the exact judgment the agent must make differently.
2. Build the relevant rows of `references/delivery-capability-map.md` from
   current official runtime documentation or safe local observation. Do not
   transfer another tool's behavior by analogy.
3. Classify each required source fragment by delivery timing: before scoped
   action, on task/method activation, or on demand. Then map that timing to an
   evidenced runtime-native surface.
4. Compare the keep-as-is path with the smallest placement or projection change.
   Name what information may be compressed, what must remain linked to its
   source, and what uncertainty blocks implementation.
5. Implement only when requested and authorized. Vendor configuration remains
   a local adapter; source content remains with its domain owner.
6. Verify the path through `commands/verify.md`. Persist the map only when
   approval, handoff, or later reproduction requires it.
