# Audit Context Delivery

Audit read-only unless the user also requests a change.

1. Name the target agent action and authoritative source before inventorying
   files or configuration.
2. Trace the current source-to-agent path through the actual runtime. Separate
   documented capability, task-specific observation, and assumption.
3. Find the highest-impact delivery failure: a governing boundary never loaded,
   detail loaded too early, an activation route that cannot be discovered, a
   stale pointer, duplicated doctrine, or a projection treated as authority.
4. State the strongest keep-as-is case and the smallest correction. Route absent
   content, artifact ownership, skill expression, and vendor mechanics to their
   respective owners instead of expanding this audit.
5. Report the target action, source, delivery path, failure, correction, and the
   observation that would show the correction unnecessary or ineffective.

Do not infer that more context is better. The relevant question is whether the
right source relation becomes available when it can change the agent's action.
