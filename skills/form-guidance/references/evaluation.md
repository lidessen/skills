# Form Guidance Evaluation

Retain only probes that can change whether the selected form is deployed.

| Probe | Question | Disconfirming observation |
|---|---|---|
| Action | Does the intended actor use the selected form to make the blocked decision? | The actor still needs internal knowledge or invents a parallel form. |
| Boundary | Does it reject a nearby request that belongs to an existing owner or needs no durable form? | It creates a skill/runtime/artifact for a one-off or routes ambiguously. |
| Handoff | Does it stop after selecting form and name the owner of the domain method? | It writes the newly selected domain's doctrine, plan, or workflow itself. |
| Authority | Does the form preserve source, verifier, and committer boundaries? | A projection becomes accepted fact, or a worker accepts its own result. |
| Context | Does the method load only the guidance needed for this judgment? | A stable skill embeds volatile project detail or misses a governing hard constraint. |

Do not use this skill as a universal alternative to generic task discipline.
Its boundary probe must show that it declines a routine task with no evidenced
form contradiction. If comparing it with another method, use the same
form-ambiguous scenario and label attribution unproven unless the baseline is
comparable.
