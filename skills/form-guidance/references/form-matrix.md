# Form Matrix

Use this only after the action gap and keep-as-is case are concrete. A form is
not justified by familiarity, polish, or an attractive name.

| Candidate | Select only when | It owns | It must not own |
|---|---|---|---|
| No new form / conversation | one actor, one bounded decision, no handoff | immediate answer | durable project memory |
| Local instruction or reference | a narrow local procedure helps an already-known actor | situational guidance | cross-session commitment or runtime state |
| Skill | an agent repeatedly needs the same judgment at activation | agent-facing method and routing | facts, acceptance, execution state |
| Durable decision artifact | a later actor must recover why a commitment or boundary exists | reviewed decision rationale | automatic execution or fact admission without verification |
| Runtime capability | behavior requires enforced tools, isolated execution, state transition, or mechanical checks | execution boundary and raw evidence | semantic acceptance or unbounded orchestration |
| Projection / CLI / dashboard | a declared source is hard to inspect or act from | access and presentation | independent fact authority |
| Bounded campaign record | one transition needs multi-session coordination, handoff, or approval | transition evidence and settlement | permanent organizational identity |

## Comparison gates

For every candidate, answer:

1. **Actor:** who uses it and what exact next decision becomes possible?
2. **Cadence:** is the need one-off, repeated, or only phase-bound?
3. **Authority:** who proposes, verifies, and commits? Could the form accept its
   own work?
4. **Lifetime:** what must survive a session, and what can be regenerated?
5. **Source:** what record is authoritative? Is this candidate a projection?
6. **Capability:** does the problem require enforced execution or only guidance?
7. **Omission risk:** what future decision fails if this form does not exist?

A solution may contain several forms only when each passes a different gate.
For example, an agent method, a human-approved decision record, an execution
evidence runtime, and a CLI review projection have different owners. A single
agent that claims all four is usually a collapsed authority boundary. This matrix
does not define the domain method inside any selected form; it ends by naming
the owner that must do so.
