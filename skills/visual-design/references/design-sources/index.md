# Design Source Index

Use this index only when host sources cannot resolve a material visual decision.
Select one to three complementary sources, then read only their detail files.
The index is fallback context, not a canon or style menu.

For actual works that expose visual choices, use `../visual-seeds/index.md`.
This index mainly supplies system and process evidence: how a relation was
formed, translated, maintained, or tested. A process reading cannot substitute
for inspecting a concrete visual artifact.

The source field has two distinct roles. Interface-system sources help with
hierarchy, layout, tokens, and maintenance. Expressive-process sources show how
a relation was discovered, translated, distributed, and tested; they are not
style presets. When a cue such as “hand-drawn,” “editorial,” or “retro” is
unresolved, read `../visual-language.md`, prefer project and primary cultural
artifacts, and use at most two expressive-process entries when they can change
a named decision.

Useful combinations:

- Content-heavy public service: GOV.UK + Apple + one local cultural source.
- Dense professional product: Linear + Primer or Carbon.
- Color/token system: Primer + Carbon; add Apple for platform adaptation.
- Cross-surface redesign: Linear + one direct host analogue + one attractive
  counterexample.
- Ambiguous expressive cue: one direct host or cultural artifact + one
  expressive-process source + one wrong-for-this-project counterexample.

For every selected source, record the observed effect, decision changed,
transfer boundary, and what must not be copied. Check the live official source
when freshness changes the decision.
