# Artifact Classes

Classify only load-bearing artifacts. The purpose is to preserve a decision,
not to produce a complete directory inventory.

| Class | Decision question | Wrong if treated as |
|---|---|---|
| **Semantic source** | Would editing this change doctrine or accepted architecture? | Projection or process note |
| **Durable evidence** | Must a later actor recover why a decision was made? | Disposable noise or semantic source |
| **Living expression** | Does this guide a repeated judgment today? | Doctrine merely because it is active |
| **Process artifact** | Does this coordinate only one bounded transition? | Permanent module |
| **Projection** | Can this be rebuilt from a declared source without losing meaning? | Independent fact authority |

## Standing gates

1. **Structure (P06):** What relation produces the visible path or duplication?
2. **Authority (P14):** If an artifact is reconstructible, keep its source named
   and authoritative.
3. **Inheritance (P12):** Preserve only the reason a later decision must recover;
   a transcript or old carrier is not automatically required.
4. **Necessity (P15):** Retain a distinction only when collapsing it would lose
   a required response, verification, recovery path, or hard constraint.

## Conditional checks

- Obtain host-designated approval when semantic authority, a public index, or
  irreversible evidence changes.
- Check index or context placement only when the transition changes discovery.
- Check fact admission only when the transition promotes a claim into shared
  authority.

Cleanup operations are derived consequences. A cosmetic tree preference does
not establish an organization gap.
