# Audit Artifact Organization

Run read-only unless the user explicitly asks to preserve a durable campaign
record.

1. Read the accepted host design and inspect the actual carrier.
2. Classify only load-bearing artifacts with
   [artifact-classes.md](../references/artifact-classes.md). Do not inventory
   every file when a class or owner answers the decision.
3. State the current role, accepted target role, and one material gap. Include
   the strongest case for leaving the carrier unchanged.
4. For each proposed distinction or move, record:
   - the future decision or response it enables;
   - the risk of omission;
   - whether an existing artifact already provides that response.
5. If no item changes a decision or protects a hard constraint, report
   `no transition warranted` and stop.
6. If a transition is warranted, propose one smallest wave, its approval
   boundary, and a falsifiable acceptance condition. Do not execute it here.

Keep the result in conversation by default. When durable handoff or approval is
needed, use the single-record format in
[artifact-paths.md](../references/artifact-paths.md).
