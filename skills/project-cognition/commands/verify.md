# Verify Project Cognition

Evaluate a cognition projection without granting it authority over its own
acceptance.

1. Name the projection, declared source revision, decisions it claims to
   support, verifier, and possible dispositions: `supported`, `repair`,
   `rebuild`, or `bypass`.
2. Select decision-changing probes before reading model conclusions. Include:
   - one downstream question whose answer crosses facet boundaries;
   - one authority or partial-failure question where a plausible summary could
     overstate the source; and
   - one invalidation question tied to an actual or hypothetical bounded change.
3. Read only the routed facets first, then inspect the source necessary to check
   their claims. Record whether the index selected sufficient context and what
   extra source had to be discovered.
4. Trace cited anchors and reconstruct a sample of incoming/outgoing relations.
   Distinguish incorrect claims, missing material relations, harmless
   compression, and open inference.
5. Check that deleting the projection would not remove authoritative project
   state, and that no Skill, Cell, generator, synthesis, projection, or consensus
   is named as the verifier or accepted its own claims. A valid verifier is an
   accountable external actor, review run, or governed process with traceable
   evidence.
6. Return findings, supported decisions, residual risk, actual evidence read,
   and one disposition. A passing verification supports only the declared
   source revision and decision field.

When claiming that this projection improves an agent's decisions, compare the
same raw task against a fresh baseline without the projection or Skill. Without
that comparison, label attribution unproven.
