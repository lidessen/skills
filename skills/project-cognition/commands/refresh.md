# Refresh Project Cognition

Refresh a retained projection from a named old revision to a named new revision.

1. Read the projection index, its declared source revision, the target sources,
   and the actual revision delta. If the old source or projection provenance is
   missing, rebuild or bypass it rather than inventing a baseline.
2. Map changed definitions and documents to their callers, consumers, state,
   authority, public contracts, migrations, tests, and cross-facet relations.
   File paths seed the impact field but do not define it.
3. Classify every existing facet as `reuse`, `refresh`, `split`, `merge`, or
   `retire`. A reuse disposition requires a concise source-backed invariance
   reason. A changed relation may affect a facet whose own files did not change.
4. Choose a delta-specific partition for affected work. Do not reuse bootstrap
   packet names unless the new impact field genuinely has the same shape.
5. Execute and source-check only the affected semantic regions, then reconnect
   them to retained facets. Preserve exact retained hashes where the carrier
   supports them; otherwise preserve an equivalent revision and disposition
   record.
6. Update the routing index, relation coverage, uncertainties, source revision,
   and invalidation signals. Keep prior raw evidence by reference; do not rewrite
   history to make the refresh look clean.
7. Test at least one downstream question that crosses a refreshed/retained
   boundary. Report missed-impact risks and whether the observed work supports
   selective refresh or instead suggests rebuilding the relevant model.

Refresh is unsupported when most decision relations cannot be traced, when
reused facets conceal changed behavior, or when maintenance repeatedly costs
more than direct source reconstruction.
