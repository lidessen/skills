# Project Cognition Concepts

## In plain words

Project cognition is a reusable, source-linked working understanding that lets
a later agent answer important project questions without rereading everything.
It is neither a transcript nor a replacement for source code and accepted
design. Its value is measured by changed downstream decisions and cheaper,
safer reconstruction.

## Essence

1. **Decision field:** the future decisions the model is meant to improve.
2. **Semantic reconstruction:** responsibilities and relations recovered from
   named project sources at the resolution those decisions need.
3. **Inheritance:** the minimum durable representation a later actor can route,
   inspect, and revise.
4. **Admission:** source-backed verification that keeps generated claims from
   becoming facts by repetition.
5. **Selective refresh:** changed-relation analysis that preserves valid prior
   work without hiding invalidated meaning.

## Domain vocabulary

- **Cognition projection:** rebuildable index and facets derived from a named
  project revision. It owns no project fact authority.
- **Decision field:** named future judgments and the source relations that can
  change them.
- **Facet:** one coherent semantic question and its source-linked answer. A
  facet name is local to the project and task, not a universal category.
- **Packet:** one bounded investigation prepared for direct or delegated
  execution. It may produce, revise, or challenge a facet.
- **Relation lens:** a way to inspect possible connections—responsibility,
  causal, state, authority, evidence, contract, change—not a required schema.
- **Coverage ledger:** a compact mapping from decision-relevant relations to
  inspected packets/facets, open boundaries, and verification status.
- **Invalidation signal:** a source change or discovered contradiction that
  requires a facet to be refreshed, split, merged, retired, or bypassed.
- **Cold evidence:** detailed run records, logs, or raw reports retained by
  reference and loaded only when verification needs them.

The project source remains the source. The cognition projection is an access
and inheritance layer whose meaning can be checked and rebuilt from that source.
